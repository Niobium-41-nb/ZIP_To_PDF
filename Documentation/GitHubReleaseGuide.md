# GitHub 发布指南

## 🚀 发布准备

### 1. 项目结构确认
确保项目包含以下目录结构：
```
项目根目录/
├── 发行版/              # 可执行文件发行包
│   ├── 压缩包转PDF工具.exe
│   ├── 快速启动.bat
│   ├── 可执行文件使用说明.md
│   ├── README.md
│   └── LICENSE
├── 源码版本/            # 干净的源代码
├── 文档/               # 项目文档
├── 开发工具/            # 开发和测试工具
└── .github/workflows/   # GitHub Actions配置
```

### 2. 版本号设置
在GitHub上创建标签来触发自动发布：
```bash
git tag v1.0.0
git push origin v1.0.0
```

## 📦 发布流程

### 方法1: 自动发布（推荐）
1. 推送版本标签到GitHub
2. GitHub Actions会自动构建和发布
3. 在Releases页面查看生成的发布包

### 方法2: 手动发布
1. 前往GitHub仓库的Releases页面
2. 点击"Draft a new release"
3. 填写版本信息：
   - **Tag version**: v1.0.0
   - **Release title**: 压缩包转PDF工具 v1.0.0
   - **Description**: 复制下面的发布说明

## 📝 发布说明模板

```markdown
# 压缩包转PDF工具 v1.0.0

## 🎉 新版本发布

### 功能特性
- ✅ **压缩包转PDF**: 支持ZIP, RAR, 7Z等格式
- ✅ **JM漫画下载**: 直接输入ID自动下载并转换
- ✅ **双模式运行**: Web界面 + 命令行模式
- ✅ **单文件可执行**: 无需安装Python或依赖

### 🚀 快速开始

#### 方法1: 简单启动
下载 `快速启动.bat` 和 `压缩包转PDF工具.exe`，双击运行批处理文件。

#### 方法2: 直接运行
```cmd
# Web界面模式
压缩包转PDF工具.exe --web

# 命令行下载JM漫画
压缩包转PDF工具.exe <漫画ID>

# 交互模式
压缩包转PDF工具.exe
```

### 📋 系统要求
- **操作系统**: Windows 7/8/10/11
- **内存**: 4GB RAM (推荐)
- **磁盘空间**: 2GB 可用空间
- **网络**: 需要网络连接（JM漫画下载功能）

### 🔧 包含文件

- `快速启动.bat` - 图形化启动菜单

- `README.md` - 项目说明文档
- `LICENSE` - MIT许可证

### 🐛 问题反馈
如遇问题，请提供：
1. 操作系统版本
2. 错误信息截图
3. 操作步骤描述

---

**温馨提示**: 请合理使用本工具，遵守相关法律法规，尊重版权。
